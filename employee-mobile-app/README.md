# Employee Mobile App - Time & Attendance

React Native/Expo mobile application for employee time tracking with photo verification.

## Features

- Employee login with auto-generated credentials
- Mandatory password change on first login
- Time In with photo capture
- Time Out with photo capture
- View today's attendance status
- View attendance history
- Secure token-based authentication

## Prerequisites

- Node.js 18+ installed
- Expo CLI installed globally: `npm install -g expo-cli`
- Expo Go app on your mobile device (for testing)
- Android Studio (for Android development) or Xcode (for iOS development)

## Setup

1. Install dependencies:
```bash
npm install
```

2. Configure environment variables:
   - Update `app.json` with your Supabase credentials:
     - `supabaseUrl`: Your Supabase project URL
     - `supabaseAnonKey`: Your Supabase anon key
     - `apiUrl`: Your Supabase Functions URL

3. Start the development server:
```bash
npm start
```

4. Run on device/emulator:
   - Scan QR code with Expo Go app (iOS/Android)
   - Press `a` for Android emulator
   - Press `i` for iOS simulator (macOS only)
   - Press `w` for web browser

## Authentication Flow

1. Employee logs in with username and password provided by admin
2. If `must_change_password` is true, redirect to password change screen
3. After password change, navigate to home screen
4. Token is stored securely using `expo-secure-store`

## Time Logging Flow

1. Employee opens app and sees today's status on home screen
2. Click "Time In" button to capture photo and log arrival time
3. Click "Time Out" button to capture photo and log departure time
4. Total hours are automatically calculated
5. View history to see past attendance records

## API Integration

The app communicates with Supabase Edge Functions:
- `POST /login` - Employee authentication
- `POST /change-password` - Password change
- `POST /time-in` - Log time in with photo
- `POST /time-out` - Log time out with photo
- `GET /my-logs` - Retrieve employee's time logs

## Camera Permissions

The app requires camera permissions for photo verification:
- iOS: Automatically requests permission on first camera use
- Android: Permissions configured in `app.json`

## Troubleshooting

### Camera not working
- Ensure camera permissions are granted in device settings
- Restart the app after granting permissions

### Login fails
- Verify your credentials with your admin
- Check that you're using the correct username (case-sensitive)
- Ensure your account is active

### API errors
- Verify `app.json` has correct Supabase URL and API URL
- Check network connection
- Contact your admin if issues persist
